全新安装

开始 ...

这是安装Zen Cart中文版的指南。如果您已有Zen Cart™而需要从旧版本升级，请看升级指南以及新增功能文档。

基本要求

您已经下载了Zen Cart中文版网上商店软件。

数据库

在继续安装前，确认您可以存取MySQL数据库，以及有相关用户名/密码。您需要用主机商提供的控制面板先建立一个数据库，如需帮助，请联系您的主机提供商，或在Zen Cart China的用户论坛提出。Zen Cart™ 无法为您创建数据库。

ZIP文件

因为您已经在阅读本指南，您应该已经将Zen Cart中文版的文件解压了。如果还没解压，现在就将文件解压，保持目录结构不变。

将Zen Cart中文版的文件上传到您的网页服务器

通过FTP上传所有文件到网站服务器的一个目录下。例如: /catalog
(我们以"/catalog"为例子。您可以"不要"目录，或使用其他名字，例如"/zencart"，或者"/store"等等)

提示: 上传文件的时候，请确认您的FTP程序及网站服务器支持"长文件名"。例如，缺省情况下，MAC OS通常不管理长文件名，您需要修改相关设置。

上传到哪个目录？

每个主机的目录结构都不相同。
很多文件用户不可见。用户可以通过浏览器存取的目录通常为:

- /home/YOURNAME/public_html
或
- /var/www/YOURNAME/httpdocs
或
- /usr/accounts/a/b/YOURNAME/httpd
等等...

如果不知道存放到哪个目录，请咨询主机商。

创建configure.php文件

在服务器上要创建两个configure.php文件，用来设定服务器参数及上传的文件的位置。创建文件后，还要修改权限。

提示: 可以通过FTP程序的chmod功能来修改权限。通常是鼠标右键点击相应目录或文件名，会弹出该选项的菜单。(可能在"属性"中)

在服务器上，找到文件: /catalog/includes/dist-configure.php
将该文件改名为 configure.php 并修改权限为 777 (所有人可以读取-写入-执行)

接下来, 在服务器上找到文件: /catalog/admin/includes/dist-configure.php
将该文件改名为 configure.php 并修改权限为 777 (所有人可以读取-写入-执行)



IIS用户提示： 如果您使用IIS，也许不熟悉"chmod 777"命令。在运行IIS的Windows系统下，您需要用鼠标右键点击文件(或者是下面说的目录), 选择[属性]，然后在[安全]标签下，确认MACHINE_NAME\IUSR_MACHINE_NAME帐号至少有"读取" 和 "写入" 的权限... 最好还有 "修改" 权限。需要分别设置列出的每一个文件/目录。(如果没有列出用户IUSR_XXXXXX，那么点击 "添加" 然后在列表中加入该用户，再设置相应权限。)

设置目录权限

现在，将下面的目录权限改为 777 (读取/写入/执行).

/catalog/cache
/catalog/images
/catalog/includes/languages/english/html_includes
/catalog/media
/catalog/pub
/catalog/admin/backups
/catalog/admin/images/graphs
提示: 打开目录catalog/images，修改所有子目录以及它们的子目录为777。例如 (这只是部分列表):
/catalog/images/attributes
/catalog/images/dvd
/catalog/images/large
/catalog/images/large/dvd
/catalog/images/medium
/catalog/images/upload
提示: 如果您忘了修改某个图像目录或其中的子目录 /images ，在使用时，会提示您无法写入该目录

其它的目录和文件，可以 CHMOD 644 或者 444, 这取决于您的服务器设置。

运行安装程序准备工作

安装程序相当智能化，应该能自动检测到下面的配置。

您需要确认这些配置是正确的，有些服务器可能会不同。

安装程序需要以下信息：
  • 全新安装的Zen Cart中文版目录的物理路径
    例如: /home2/zencart/public_html/catalog


  • HTTP 路径(您的商店域名及目录的网址)
    例如: http://www.mydomain.com/catalog

SSL

如果您安装正式的运营网店，最好要用SSL保证客户交易数据的安全。如果您还没有SSL证书，可以向主机商咨询。你需要在下面输入SSL的资料，通常已经是正确的，除非您使用"共享SSL"服务。您的主机商会提供正确的网址。
如果是在本地或测试服务器上安装，不需要保护敏感数据时，可以不修改SSL设置，在安装时不用打开。
  • HTTPS服务器 (您的网站的SSL网址)
    例如: https://www.mydomain.com
    提示: 如果您使用共享SSL，该地址类似于:
    https://mydomain.secureservername.net/
    - 或者 - https://secure.sharedservername.net/~username

  • HTTPS路径(您的商店的SSL域名及目录的网址)
    例如: https://www.mydomain.com/catalog
    - 或者 - https://secure.sharedservername.net/~username/catalog

运行安装程序

在浏览器地址栏，输入您新商店的网址, 安装程序将自动运行。
例如: http://www.mydomain.com/catalog
- 或者 - 直接运行安装程序: http://www.mydomain.com/catalog/zc_install

如果您看到的是文件名和目录的列表，请与您的网站服务商联系，了解如何让服务器自动检测PHP文件。

您会看到"Zen Cart中文版"的安装页面，上面列出了Zen Cart中文版的功能。
点击继续将转到许可协议页面，您需要阅读并接受GPL许可协议。

系统检查

接着，安装程序会检测服务器是否满足运行Zen Cart中文版的技术要求，列出需要您确认的主机参数。所有标记为红色的或带 "X"的，都需要修正，安装程序才能继续。标记为橙色或黄色的 "注意"项目，是警告信息，与安装不一定有关。图像目录及之前说明的内容也会列出。如果您在服务器上做了修改，可以点击重新检查或在浏览器下按F5键，刷新显示，检查所作的修改。

如果在服务器上发现旧版本的Zen Cart，安装程序会检测数据库补丁版本并显示在屏幕上。在这个情况下，屏幕上会显示"升级"按钮让您选择。查看升级指南。

一旦您对"安装预查"满意后， 安装检查就完成了(理想状况是, 所有标记都是绿色), 您可以点击屏幕下角的"安装"按钮。

系统设置

在系统设置页面，您需要输入上面栏目"运行安装程序准备工作"中提到的内容。

指明是否在服务器上使用SSL(在登录、结帐及管理页面使用安全连接)。如果您还没有SSL证书，现在不要打开该选项，以后可以修改。(见Zen Cart China 网站上的常见问答)。

提示: 如果出现以下错误，检查以上步骤，保证没有遗漏。所有的错误信息都带有弹出窗口说明，可以点击详细说明...链接。
警告: 发现问题
  • /includes/configure.php 文件不存在。  详细说明...
  • /admin/includes/configure.php 文件不存在。  详细说明...

phpBB安装

下面，提示您输入phpBB路径。如果您的网站上已经安装了phpBB论坛，可以和Zen Cart连接，只要输入服务器上安装phpBB文件的路径。例如, 如果在......public_html/forums 中，那么您在phpBB文件路径中输入/forums。

数据库安装

下一个屏幕, 需要输入MySQL数据库的信息，包括用户名和密码。这可以从您的主机商提供的cPanel或相关屏幕获得。如果您还没有MySQL数据库及用户名和密码，那么您需要创建一个。
具体方法请咨询您的主机提供商。在Zen Cart中文版安装程序能继续安装前，您需要创建数据库和用户。.

该屏幕上的其它信息:
  • 目前，MySQL是主要的数据库类型。
    以后的版本也许会支持其它的数据库类型。

  • 出于安全考虑，建议您将数据库Sessions保存在数据库中。

商店设置

现在，填写您的商店信息。

提示: 除了"演示数据"，这里所有的信息，以后在商店的管理页面都可以修改。

演示数据
如果您想安装演示数据，选择 [是]。

建议您安装演示数据以熟悉本软件。演示数据内含很多例子，解释和展示了Zen Cart的大量功能。

您也可以安装一个带演示数据的测试站点，及一个单独的正式站点。这样，可以利用演示数据获得帮助。


点击保存商店设置后，会有些延迟，是因为要创建数据库表及安装可选的演示数据。

管理设置

现在，完成管理信息，包括设置管理员登录名、管理员邮件及密码。

提示: 用户名和密码都是大小写敏感的。

保存管理设置，安装完成!
如果安装到这里都没有错误，您现在就可以进入管理页面或商店前台。

安装之后

进入商店前台后，您会看到configure.php文件和/zc_install目录的安全警告信息。


configure.php文件
您现在要修改两个configure.php文件的权限，chmod 644 ( 或444， 与您的服务器有关...有时不能通过FTP来设置444，而要使用主机商提供的控制面板或文件管理器来设置)

这两个文件位于:
/catalog/includes/configure.php
/catalog/admin/includes/configure.php

最好在安装完成之后，从服务器上下载一份这两个文件的备份，因为这些文件在安装时，已经按您的服务器设置好了。

如果出现错误或问题，经常可以通过修改这两个文件来纠正。


zc_install目录
最后, 要删除或改名 /catalog/zc_install 目录，例如: /catalog/zc_install_complete

提示: 不要 使用zc_install_complete ，因为黑客读了本说明，也会知道这个名字。
